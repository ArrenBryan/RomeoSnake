- Multiple Source Code dengan 1 Source Code kurang lebih logika-nya sama, hanya saja structure code untuk Multiple Source Code dengan konsep semi-OOP. Mengapa kita sebut semi-OOP, karena kita juga masih belum terlalu paham / belum punya pengalaman bagaimana cara OOP di bahasa C, referensi kita belajar OOP di bahasa Java. Maka dari itu, kita hanya memisahkan beberapa yang dapat di jadikan object sebagai header yang terpisah (tidak sepenuhnya OOP).

- Di Multiple Source Code, belum ada fitur game yang menggunakan obstacle (penghalang seperti pagar). Maka dari itu, di Multiple Source Code belum bisa mengakses game classic level Medium, Hard ataupun Expert. Disisi lain untuk 1 Source Code, kami sudah adakan fitur game yang menggunakan obstacle (penghalang seperti pagar) di source code itu.

- Jadi dapat dikatakan, untuk full version progress pengerjaan, 1 Source Code lebih banyak fitur-nya dibanding Multiple Source Code karena Multiple Source Code hanya baru dapat mengakses game classic level Easy. (ada beberapa fitur dari 1 Source Code belum dipindahkan ke Multiple Source Code karena beberapa kendala)

- Mengapa kami membuat dua konsep structure code yang berbeda, karena menurut kami pada 1 Source Code yang sama, code pun jadi semakin sulit untuk di baca dan untuk merapikan-nya (clean code) pun sulit untuk dilakukan karena pada dasar-nya 1 Source Code yang kami buat itu hanya untuk prototyping.
Maka dari itu, kami membuat Multiple Source Code dengan konsep semi-OOP ini.
